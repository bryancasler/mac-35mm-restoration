# ANGLE 3 — How professional restoration tools do automatic dust-busting

## Per-tool intelligence

**Filmworkz Phoenix / DVO (ex Digital Vision — the deepest documented)**
- DVO Dust is built on PHAME ("Predictive Hierarchical Adaptive Motion Estimation", Emmy-awarded) — full motion-compensated temporal detection + concealment; marketing claim: ~90% of visible imperfections removed without artifacts, effective even on low-contrast defects (mold/lichen). https://filmworkz.com/dvo/dust/
- Documented controls (from DVO Dust GT user guide, surfaced via search snippets; PDF at https://filmworkz.com/wp-content/uploads/2024/06/DVO-Dust-GT-UG-.pdf currently 404s but text is indexed):
  - **Motion slider 0–256**: 0 = maximum motion sensitivity (safest — temporal filtering suppressed wherever motion detected); 256 = no motion adaptation (temporal filtering applied fully in moving areas). Workflow: start at 0, raise stepwise until moving areas get enough filtering.
  - **Motion estimation toggle**: with ME enabled, "much more aggressive settings can be used without introducing smearing" — i.e., detection thresholds and motion-adaptation are deliberately coupled to ME availability.
  - **Temporal adaptation + trim size**: falsely-detected dirt is excluded by comparing the current frame's detected-dirt map with the previous frame's within a window ("trim size"); smaller window = more filtering. This is dirt-persistence rejection: real dirt is single-frame.
  - **MDC toggle**: compare against prev+next only, vs prev+current+next, for higher sensitivity.
  - Presets Extra Light → Extra Heavy as entry points.
- DVO Dry Clean (newer auto tool): separate **Dark / Bright object detection** switches, **Dust Size Limiter** ("focus on smaller blemishes only — reducing false detections"), **Smoothness** (blend concealment edges), **Temporal Safety** ("prevents false positives where unexpected objects appear or disappear" — the birds problem, as an explicit user-facing safety system), **Cut Safety** (raises safety near scene cuts), and a "force processing of detected dust even if considered unsafe" override — i.e., detection and a safety classifier are separate stages. https://filmworkz.com/dvo/dry-clean/
- Tool split mirrors defect size: Dust/Dry Clean = automatic small-defect; DVO Fix = "incredible dust & artifact removal" for larger single defects (https://filmworkz.com/dvo/fix/); scratch handled by a different tool entirely (persistent defects break the temporal-impulse assumption).

**DaVinci Resolve (Studio) — Automatic Dirt Removal + Dust Buster**
- ADR: optical-flow based, explicitly scoped to defects "that last for one or two frames and then disappear"; repairs "while maintaining structurally consistent detail". Controls: **Motion Estimation Type** (None/Faster/Normal/Better), **Neighbor Frames** (more frames = more dirt found), **Repair Strength** (aggressiveness), **Dirt Size Threshold** (tune above grain size), **Show Repair Mask** (QC the detection independently of the repair). Documented failure modes: static vertical scratches and full-shot lens dirt are out of scope. https://www.steakunderwater.com/VFXPedia/__man/Resolve18-6/DaVinciResolve18_Manual_files/part3281.htm
- Dust Buster (guided): **Auto patch = 5-frame window (±2)**, "sampling the best of these five frames", and it "prioritizes images two frames away to avoid the appearance of frozen grain, but only if they're a suitable match" — a concrete anti-ghosting/anti-frozen-grain heuristic worth copying verbatim. https://filmmakingelements.com/remove-dirt-and-dust-in-davinci-resolve/ , https://www.premiumbeat.com/blog/resolve-15-dirt-repair-dust-buster/

**MTI DRS Nova**
- **AutoFilter** (classic auto dirt) has **min/max size** controls; **Shine** is their newer detector, claimed to "distinguish unintentional artifacts from authentic film elements" and (v7) adds **grain suppression control** to improve detection on grainy stock — i.e., pre-suppress grain before the temporal outlier test. https://mtifilm.com/software/drs-nova/
- **Camera Motion analysis**: analyzes each shot, produces "motion segments" classed **Static / Low / High**, and the artist assigns different filter aggressiveness per segment (supported by Shine). Per-shot motion-adaptive parameterization is a first-class workflow concept. https://mtifilm.com/software/drs-nova/
- Gate Hair tool = camera-motion analysis + inpainting + temporal tracking for persistent defects. v6 AI = MTai FrameGen (generative frame re-creation for slugs/jump cuts), not the dirt detector. https://postperspective.com/mti-films-drsnova-v6-0-with-ai-based-image-cleanup/ ; user guide PDF (image-based, not text-extractable): https://mtifilm.com/wp-content/uploads/2024/06/DRS-Nova-User-Guide_v6_2024.pdf

**DIAMANT (HS-ART) — Joanneum Research heritage**
- Core algorithms come from Joanneum Research (projects LIMELIGHT/FRAME/DIAMANT); founding paper: Schallauer, Pinz, Haas, "Automatic Restoration Algorithms for 35mm Film" (Videre 1999) — dust/dirt detection designed to be robust in noise, with detection and removal **separated into a two-step approach so the expert can review intermediate analysis data** before removal. https://www.researchgate.net/publication/2430761_Automatic_Restoration_Algorithms_for_35_mm_Film , https://www.hs-art.com/index.php/about-us-2/partners
- Product: automatic dust filters + DustBuster+ interactive "click & fix" with **ExInpaint** (exemplar-based inpainting) for hairs/large defects — spatial inpaint as manual fallback where temporal fails. https://www.hs-art.com/index.php/solutions/dustbusterplus , https://www.hs-art.com/index.php/tutorials/video-tutorials-overview/82-dustbuster-exinpaint-hair-removal
- Current "AI dust & dirt" engine; their Defect.AI research project targets training defect-detection models with ~10% of normal labeled-data volume by "capturing expert knowledge embedded in existing sophisticated hand-crafted detectors" (i.e., classic detectors bootstrap/label training data for the net). https://www.hs-art.com/index.php/research-main/deeprestore-menu

**PFClean (The Pixel Farm)**
- Auto/hybrid/manual detection with adjustable sensitivity/threshold; all fixes stored as **per-frame metadata maps "right down to individual dust specks"** — non-destructive detect-then-commit architecture. https://www.pfclean.com/

**Snell & Wilcox Archangel Ph.C (real-time hardware)**
- Dirt concealment built on **phase-correlation motion estimation**; same vectors reused for weave/stabilization, MC noise reduction, and scratch/dirt concealment. https://www.thameside.tv/documents/Archangel_PhC.pdf , https://patents.justia.com/assignee/snell-wilcox-ltd

**Patent gold: BBC US5978047 "Blemish concealment in video signals" (the most implementable single document found)**
https://patents.google.com/patent/US5978047A/en
- Stage 1: motion-compensated **3-frame median** (prev/next shifted by measured motion vectors) = preliminary concealment.
- Stage 2: |median_output − input| → LPF → threshold t1 → binary candidate flags → **dilate flags 1+ px**.
- Stage 3, three motion safeguards (this is the anti-birds machinery):
  1. **Adjacent-frame flag suppression**: if a flag exists in the prev/next frame within a set search radius of a current-frame flag, kill the current flag (dirt is temporally isolated; a moving object flags in consecutive frames along its path).
  2. **Size-dependent search radii**: flags split into small/medium/large groups; larger flags get larger adjacency search distances.
  3. **Edge-steepness test**: dirt has hard edges; real moving objects have motion-blurred (fuzzy) edges at ~180° shutter. Measure candidate extent at two thresholds t3/t4 of the difference signal; a large extent difference = soft edge = motion → reject.
- Stage 4: per-pixel selector — median output only under surviving flags, source passthrough elsewhere.

**Patent: Thomson US9167219 "Method and apparatus for automatic film restoration"**
https://patents.google.com/patent/US9167219B2/en
- Metadata-driven 3-phase workflow: detect → human QC on metadata → remove. Detection = noise-level estimation → spurious-pixel detection → **clustering into objects → per-object weighting → weight threshold**. **Dual thresholds**: low detection threshold (catch everything) + high removal threshold (only confidently remove). Explicit "pathological motion" handling. Per-artifact metadata: bbox, pixel mask, weight, cross-frame links.

**Academic canon (the algorithms the above products are built from)**
- **ROD** (rank-ordered differences, Nadenau & Mitra): from 6 MC pixels (3 prev, 3 next), compute rank-ordered differences to current pixel; blotch if ROD_k > T_k for any of three thresholds T1<T2<T3 (T1 most sensitive). https://www.researchgate.net/publication/2910284_B1_Blotch_and_Scratch_Detection_in_Image_Sequences_based_on_Rank_Ordered_Differences
- **SDIp/sROD** variants and comparative surveys: https://scispace.com/pdf/blotch-detection-and-removal-for-archive-video-restoration-133t0enm9j.pdf (5-frame multiresolution block matching + ROD + 3D vector median interpolation, vs 3-frame baseline); https://link.springer.com/article/10.1007/s11265-014-0942-8
- **Pathological motion**: Corrigan, Harte, Kokaram — 5-frame temporal profile distinguishes the **impulsive** profile of a blotch from the **quasi-periodic** profile of erratic motion; plus **motion-field divergence** as a spatial PM indicator; PM regions are excluded from removal. This paper is the direct academic answer to "hit and miss" false positives. https://www.researchgate.net/publication/224057430_Pathological_Motion_Detection_for_Robust_Missing_Data_Treatment_in_Degraded_Archived_Media
- Complementary-detector fusion (combine ROD + SDI + spatial detectors, vote): https://www.researchgate.net/publication/262405910_Fusion_of_complementary_detectors_for_improving_blotch_detection_in_digitized_films ; segmentation-assisted dirt detection: https://www.researchgate.net/publication/221259743_Segmentation-Assisted_Dirt_Detection_for_the_Restoration_of_Archived_Films
- ML route: RTN "Bringing Old Films Back to Life" (CVPR 2022; recurrent transformer, flow-guided alignment, **unsupervised defect localization** by contrasting current frame vs temporally-accumulated hidden state; code public) https://arxiv.org/abs/2203.17276 ; DeepRemaster https://arxiv.org/pdf/2009.08692 ; Correl-Net defect segmentation https://link.springer.com/chapter/10.1007/978-3-031-31438-4_17 ; GAN dust/scratch removal https://arxiv.org/pdf/2009.10663 ; synthetic-degradation training pipeline + benchmark https://arxiv.org/pdf/2607.02131
- FOSS baseline reality check: RemoveDirt restores **only motion blocks** and leaves static areas untouched (https://github.com/pinterf/RemoveDirt); DeSpot's `seg` modes remove only spots with **no overlap with motion zones** (http://avisynth.nl/index.php/DeSpot); users report the classic ball/bird disappearance and resort to masks (https://forum.videohelp.com/threads/407398-problems-with-avisynth-dirt-removal-very-basic). Our current chain sits at this baseline — the pro tools differ mainly in the layers below.

## The playbook — ranked by expected impact on "hit and miss"

1. **Split detection from concealment; produce a reviewable mask.** Every pro tool does detect → mask → (QC) → fill; none does blind in-place filtering like RemoveDirt/SpotLess. Gives you: Show-Repair-Mask debugging (Resolve), dual thresholds (detect low / remove high — Thomson US9167219), per-blob decisions, and bit-exact passthrough of clean pixels (BBC US5978047 selector stage). Implement as: candidate mask in `frame_eval` (numpy) → blob filtering → `MaskedMerge` of a fill clip.
2. **Use 5 frames (±2, motion-compensated), not 3.** Dirt is impulsive in exactly one frame; erratic real motion is not. 5-frame temporal profiles are the single biggest false-positive killer (Corrigan/Kokaram PM paper; Resolve Dust Buster ±2 "best of five"; survey's 5-frame ROD pipeline). We already have MVTools Compensate — add ±2 compensation and require: |cur−prev1|,|cur−next1| both large AND same sign, while prev1↔next1, prev2↔next2 agree with each other.
3. **Implement the BBC US5978047 motion-rejection triad** on candidate blobs: (a) adjacent-frame flag suppression within a size-dependent radius, (b) small/medium/large blob classes with different rules, (c) edge-steepness test (sharp edge = dirt, motion-blurred edge = moving object). All three are cheap numpy/OpenCV ops on masks and are the literal patented answer to the birds problem.
4. **Rank-order detection instead of plain absolute difference.** ROD/sROD over 6 MC neighbors with a threshold ladder (T1<T2<T3) is far more robust to grain and slight misalignment than the Clense-style min/max logic in RemoveDirt. Directly codable in `frame_eval`.
5. **Blob-level size gating and per-object weighting.** Connected-components on the candidate mask; enforce min size (reject grain — Resolve Dirt Size Threshold, MTI grain suppression) and max size (reject occlusions/objects — MTI AutoFilter min/max, DVO Dust Size Limiter); weight blobs by contrast/steepness/temporal isolation and only remove above the removal threshold (Thomson patent).
6. **Motion-adaptive aggressiveness.** DVO's Motion 0–256 blend and MTI's per-shot Static/Low/High motion segments: compute a per-block motion magnitude (we have MVTools vectors), then scale detection sensitivity down (or disable removal) as local motion rises; optionally classify whole shots and parameterize per shot.
7. **Temporal + cut safety systems.** DVO Dry Clean's named safeties: protect objects that appear/disappear (temporal safety = adjacent-frame suppression at blob level), and raise thresholds within N frames of a scene cut where MC breaks (cut safety — we already have scene detection via ffmpeg/`SCDetect`).
8. **Conceal with MC temporal fill, prioritized away from frame ±1; spatial inpaint fallback.** Fill = median of MC(prev,next) (BBC), or copy best-matching of ±2 preferring the 2-away frame to avoid frozen grain (Resolve Dust Buster), with match-quality check; where MC confidence is low or blob is large, fall back to exemplar inpainting (DIAMANT ExInpaint; `cv2.inpaint` TELEA or a patch-match port) instead of ghosting.
9. **Pathological-motion map as a protection channel.** Motion-field divergence + 5-frame profile classification to mark regions where temporal logic must not fire (Corrigan et al.). MVTools gives us the vector field; divergence is a numpy gradient.
10. **Detector fusion + grain pre-suppression.** Run detection on a lightly denoised copy (grain suppression before the outlier test — MTI Shine v7), and AND/vote multiple detectors (ROD + steepness + isolation) rather than trusting one (fusion paper).
11. **ML tier (optional, biggest lift):** either a defect-segmentation net feeding principle 1's mask path (Correl-Net; HS-ART Defect.AI's approach of using classic detectors to bootstrap labels), or end-to-end RTN (public code, CoreML/ONNX-able) — with synthetic degradation pipelines (arXiv 2607.02131, 2009.10663) to train on our own scans.

Cross-cutting observation: no pro tool tries to make one pass handle everything — persistent defects (scratches, gate hairs, lens dirt) are always separate tools from single-frame dirt, and all pro auto-dust tools explicitly declare static defects out of scope (Resolve manual). Our DeScratch/RemoveDirt split matches the pros; what is missing is the detection/masking layer (principles 1–7), which is exactly where "hit and miss" lives.
