ANGLE 4 FINDINGS — Custom detect-then-inpaint pipeline: implementable design

BOTTOM LINE: Feasible with near-zero new compiled code. The detector (a) is a solved problem in the literature (Kokaram SDIa/ROD family) and can be expressed entirely in native VS `std.Expr` over `[cur, compP, compN]` — no numpy needed for the pixel test. Shape filtering (b) is the only piece that wants numpy/OpenCV (`cv2.connectedComponentsWithStats`, ~1–3 ms/frame). Inpainting (c) should be MC-borrow via `MaskedMerge` (native, free) with `cv2.inpaint` Telea as fallback for the rare both-frames-dirty case. Two large shortcuts exist: the **vapoursynth org's DeSpot port** (compiled detector with exactly the shape/motion logic we'd write) and **SpotDelta in Selur's scripts** (a pure-VS detect+protect+repair script we can crib mask machinery from).

---

(a) MOTION-COMPENSATED TEMPORAL OUTLIER DETECTION — prior art + formulas

The proposed detector ("both MC diffs high AND cross-diff low") is exactly the classic blotch-detector family:

- **SDIa (Spike Detection Index, Kokaram & Rayner 1993):** blotch at x if `min(|I_t(x) − I_MCprev(x)|, |I_t(x) − I_MCnext(x)|) > T`. High detection rate, many false alarms — which is why everyone adds the cross-check and shape stage. ([survey/fusion paper](https://perso.telecom-paristech.fr/bloch/papers/PRL-Sorin2007.pdf), [Kocaeli blotch paper](http://kulis.kocaeli.edu.tr/pub/aeu08_blotch.pdf), [ResearchGate overview](https://www.researchgate.net/publication/221579513_Blotch_Detection_and_Removal_for_Archive_Video_Restoration))
- **ROD (Rank-Ordered Differences, Nadenau & Mitra 1997):** take 6 reference pixels from MC-prev and MC-next (co-located + vertical neighbors, 3 per frame), rank r1≤…≤r6; if pixel above median use RODk = I − r_(7−k), below median RODk = r_k − I; blotch if ANY RODk > Tk. The fusion paper reports thresholds on the order of T1≈5, T2≈10, T3≈20 (8-bit units) — treat as starting points, tune on-box. **sROD** simplifies to: blotch if I is outside [min(refs)−T, max(refs)+T].
- **DeSpot (Fizick, AviSynth)** is the production-grade engineered version: p1=24 (outlier contrast), p2=12 (spot-growing threshold), mthres=16 (motion), `motpn=true` = motion computed **prev-vs-next** — literally the `|comp_prev − comp_next| low` guard (uncompensated in DeSpot; ours would be compensated, strictly better). Plus `ranked=true` (min/max of 3+3 temporal neighbors — a mini-ROD), `sign` (dark-only/bright-only polarity — 35mm positive dirt is dark, negative-scan dirt is bright: expose this), `mscene=40` scene-change kill-switch, `merode` motion-map erosion. Full parameter semantics documented at [avisynth.org.ru/despot](https://avisynth.org.ru/despot/despot.html) and [avisynth.nl/DeSpot](http://avisynth.nl/index.php/DeSpot).

**VS-native implementation (no numpy):** with `compP = mv.Compensate(cur, super, vecP)`, `compN = mv.Compensate(cur, super, vecN)` (already in our SpotLess chain):

```
mask = core.akarin.Expr([cur, compP, compN],
    "x y - abs T1 > x z - abs T1 > and  y z - abs T2 <  and  x y - x z - xor not and  peak 0 ?")
```
i.e. both errors > T1 (≈20–28 8-bit), cross-error < T2 (≈10–14), and same sign of deviation (a spike, not a crossing edge). Starting anchors: DeSpot's p1=24 / mthres=16. Add polarity variant (`x y - 0 <` for dark dirt only). Guards that cost nothing: `mv.SCDetection` frame props to zero the mask at scene changes, and `mv.Mask(kind=1)` (SAD mask) from [vs-mvtools](https://github.com/dubhater/vapoursynth-mvtools/blob/master/readme.rst) to suppress detection where motion estimation itself failed (occlusions — the main false-alarm source). Cost: Expr is a compiled per-pixel op, <1 ms/frame; the Compensate calls we already pay for.

Status: **needs writing, but only ~30 lines of .vpy; every building block exists on arm64 today.**

(b) MASK POST-PROCESSING / SHAPE CLASSIFICATION

Two tiers:

1. **Native VS morphology (cheap, no Python):** `std.Minimum`×k + `misc.Hysteresis` = "remove components smaller than k while restoring survivors to full extent" — exactly the pattern used by SpotDelta's `_remove_small_spots` ([SpotLess.py](https://github.com/Selur/VapoursynthScriptsInHybrid/blob/master/SpotLess.py)). `std.Maximum`×n + BoxBlur = dilate+feather for the merge mask (DeSpot's `dilate=1, blur=1` equivalents). This alone kills grain-sized false positives.
2. **numpy CC shape stats (for dust-vs-scratch-vs-motion triage):** `cv2.connectedComponentsWithStats` on the 1440×1080 uint8 mask → per-component area, bbox, centroid. Classify: **scratch** if bbox_h/bbox_w ≥ ~5 and bbox_h > ~40 px and fill-ratio high (route to DeScratch/spatial path); **dust** if 4 ≤ area ≤ ~500 px and aspect < 3 (route to MC-borrow); **suspected motion** if area > ~2000 px or touching a high-SAD region (drop — protects the "hit and miss" failure mode where RemoveDirt eats real motion). This is DeSpot's `pwidth/pheight/maxpts/p1percent/seg` logic reimplemented with full control. Cost: OpenCV's CC (SAUF/BBDT) on 1.5 MP is ~1–3 ms on an M4 P-core (estimate — benchmark on-box; `scipy.ndimage.label` is ~3× slower, avoid). Entirely feasible in a per-frame Python callback at 24 fps offline speeds.

Status: tier 1 **exists** (native filters, arm64 fine — needs `misc` a.k.a. vs-miscfilters-obsolete, in vs-plugin-build ecosystem). Tier 2 **needs writing** (~60 lines numpy).

(c) INPAINTING THE MASKED REGIONS

- **Primary: MC-borrow (recommended).** `fixed = MaskedMerge(cur, repair_src, mask_dilated)` where `repair_src = Expr([cur, compP, compN], "x y - abs x z - abs < y z ?")` (borrow from whichever compensated neighbor agrees worse with the defect, i.e. lower cross-error side) or simply `zsmooth.Clense`-style median of (compP, cur, compN) — all native, zero extra generations, grain-preserving outside the mask. Handles dust (1-frame transients) essentially perfectly; this is what RemoveDirt/DeSpot do internally, but doing it explicitly gives us the mask for QC overlays.
- **Fallback: spatial cv2.inpaint** for pixels where both compensated references are themselves invalid (persistent scratches, 2-frame dirt, occlusion): `cv2.inpaint(luma_u8, mask_u8, 3, cv2.INPAINT_TELEA)`. 8-bit, single-channel; cost scales with masked-pixel count — for realistic dirt masks (<0.5% of frame) expect single-digit ms; NS is comparable speed and per [LearnOpenCV's comparison](https://learnopencv.com/image-inpainting-with-opencv-c-python/) often looks better; Telea better for thin regions with small radius. opencv-contrib adds `xphoto` FSR/Shiftmap (better quality, much slower).
- **VS-native plugins:** [dnjulek/VapourSynth-cv_inpaint](https://github.com/dnjulek/VapourSynth-cv_inpaint) wraps Telea/NS/FSR/Shiftmap but is 8-bit RGB/GRAY only, one 2022 release, no macOS arm64 binary, and drags in an OpenCV C++ dependency — calling cv2 from Python is strictly easier for us. [invisiblearts/VapourSynth-Inpaint](https://github.com/invisiblearts/VapourSynth-Inpaint) is abandoned. **No maintained arm64 VS inpaint plugin exists.**
- **ML option (later):** [dan64/vs-propainter](https://github.com/dan64/vs-propainter) — ProPainter video inpainting inside VS, MIT, actively maintained (June 2026 release), PyTorch ≥2.2, mask-clip API. CUDA-oriented; MPS support unverified — treat as a research spike, not the plan of record. Also note [Spotless-Film](https://github.com/yahyarahhawi/Spotless-Film) (U-Net dust detect + LaMa inpaint for still film scans) as a model source if we ever train a detector.

Status: MC-borrow **exists** (native). cv2 fallback **needs writing** (~20 lines). 

(d) NUMPY/OPENCV INTEROP IN VAPOURSYNTH — exact pattern + cost

- Canonical pattern ([muvsfunc wiki "OpenCV Python for VapourSynth"](https://github.com/WolframRhodium/muvsfunc/wiki/OpenCV-Python-for-VapourSynth), [doom9 thread](https://forum.doom9.org/archive/index.php/t-184594.html), [VS Python reference](https://www.vapoursynth.com/doc/pythonreference.html)): use `core.std.ModifyFrame(clip=cur, clips=[cur, compP, compN], selector=fn)`; inside `fn(n, f)`: `fout = f[0].copy()` (writable), zero-copy reads `a = np.asarray(f[1][0])` (API4: `frame[plane]` is the supported array wrapper; `get_read_array` is the deprecated API3 spelling), write with `np.asarray(fout[0])[:] = result`, return `fout`. Wolfram's `muvsfunc_numpy.numpy_process` wraps this but says "temporal filters not supported" — irrelevant for us: passing the *compensated clips as separate clips at the same frame index* turns our temporal problem into a spatial one, which ModifyFrame handles.
- Overhead at 1440×1080: reads are zero-copy; the writable frame copy is 1.5 MB (8-bit luma) / 3 MB (16-bit) ≈ 0.1–0.3 ms; Python callback + GIL is the real cost. Realistic budget for detect-mask-CC-classify in Python: 3–6 ms/frame; add cv2.inpaint only on flagged frames. Expect the Python stage to hold >100 fps standalone, so it won't be the chain bottleneck (MVTools Analyse is). Caveat: VS calls Python filters under the GIL, serializing that stage across worker threads; cv2 releases the GIL internally for big ops, and keeping the pixel-test in native Expr (per (a)) keeps the Python stage small.
- **Install next to Homebrew VS Python 3.14:** Homebrew Python is PEP 668 externally-managed — do NOT `--break-system-packages`. opencv-python moved to **cp37-abi3 wheels** in the 5.x line: `opencv_python-5.0.0.93-cp37-abi3-macosx_13_0_arm64.whl` (uploaded 2026-07-02) works on Python 3.14/arm64 (the earlier "no 3.14 wheels" [issue #1155](https://github.com/opencv/opencv-python/issues/1155) predates the abi3 move; verify with a pip dry-run). Strategy consistent with ADR-6's app-managed-dir approach: `python3.14 -m pip install --target <app-managed>/pysite numpy opencv-python-headless` and prepend to `PYTHONPATH` when launching vspipe. Use **headless** (no GUI frameworks). ([opencv-python PyPI](https://pypi.org/project/opencv-python/), [pip install docs](https://docs.opencv.org/4.13.0/db/dd1/tutorial_py_pip_install.html))

(e) EXISTING IMPLEMENTATIONS TO REUSE

1. **[github.com/vapoursynth/despot](https://github.com/vapoursynth/despot)** — Fizick's DeSpot as a combined VapourSynth+AviSynth C++ plugin, under the official vapoursynth org (pushed Nov 2023; src/ + msvc/ only, no meson — needs a trivial Makefile; plain scalar C++, should compile arm64 cleanly). This is the entire (a)+(b) detector with 20 years of tuning, compiled. GPL-2.0 — **check GPL-2-only vs our GPL-3 compatibility before shipping**. Highest-leverage single item found in this angle.
2. **[Selur/VapoursynthScriptsInHybrid/SpotLess.py — SpotDelta](https://github.com/Selur/VapoursynthScriptsInHybrid/blob/master/SpotLess.py)** — VS port of chmars' AviSynth SpotLessDelta ([doom9 origin](https://forum.doom9.org/showthread.php?p=1946031)): runs SpotLess, then builds delta masks (source−filtered), erode+Hysteresis small-spot removal, Scharr edge protection, and restores detail SpotLess wrongly removed. Pure native VS (mvtools + zsmooth/tmedian + misc/hysteresis + optional akarin) — directly addresses our "hit and miss" complaint (it un-deletes false positives) and is a ready-made library of mask ops for the custom pipeline. **Exists; test first before writing anything.**
3. [lostfunc](https://github.com/theChaosCoder/lostfunc) — includes Didée's minimal `DeSpot` prefilter; minor prior art. [muvsfunc_numpy](https://github.com/WolframRhodium/muvsfunc/wiki/OpenCV-Python-for-VapourSynth) — interop helper. John Meyer's doom9 workflow (motion-compensated DeSpot + RemoveDirt + MDegrain, [thread](https://forum.doom9.org/archive/index.php/t-144271-p-5.html), [thread 2](http://forum.doom9.org/archive/index.php/t-165975.html)) — validates the architecture: detector + MC repair + protective gating is what the best hobbyist restorations converged on.

SUGGESTED BUILD ORDER (per-frame budget @1440×1080, M4 Pro): (1) try SpotDelta as-is (0 new code); (2) explicit-mask pipeline: Expr detector <1 ms + native morphology <1 ms + MaskedMerge MC-borrow <1 ms — all native, ships without Python interop; (3) add numpy CC classifier (~2–4 ms) routing scratch-shaped components to DeScratch and vetoing motion-shaped ones; (4) add cv2.inpaintTelea fallback (~5–10 ms on affected frames only); (5) optionally build vapoursynth/despot for arm64 as a compiled alternative A/B. Pieces needing writing: the ~150-line .vpy glue for (2)–(4) and a Makefile for (5); everything else exists.
